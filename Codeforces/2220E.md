# 2220E - Coloring a Red Black Tree

## Problem Summary
Given a tree with n nodes, some colored red (value 1) and some black (value 0), we perform operations to color all nodes red. Each operation:
1. Select any node a
2. Choose a random neighbor b uniformly
3. Recolor node a with color of b

Goal: Calculate the minimum expected operations to color all nodes red using optimal strategy.

## Problem Analysis

### Expected Value Calculation
For a black node with:
- Total degree: d (number of neighbors)
- Red/reachable neighbors: r (neighbors that are or can become red)

Expected operations to color it red: **d / r**

This comes from geometric distribution: if we repeat random selection with success probability r/d, expected trials = d/r.

### Algorithm Approach
1. **Distance Calculation**: Use BFS from all red nodes to find distance from each node to nearest red node
2. **Neighbor Classification**: For each black node, identify "helping neighbors":
   - Red neighbors (distance 0)
   - Neighbors closer to red nodes (distance < current node's distance)
3. **Expected Cost**: For each black node, add (degree / helping_neighbors) to total

**Time Complexity**: O(n) for BFS + O(n*d) for calculations = O(n)
**Space Complexity**: O(n)

## Solution Approach

### Phase 1: BFS Distance Calculation
```cpp
queue<int> bfsQueue
vector<int> distanceFromRed(n+1, -1)

// Initialize all red nodes with distance 0
for each red node:
  queue.push(node)
  distanceFromRed[node] = 0

// BFS to find distances
while queue not empty:
  current = queue.pop()
  for each neighbor of current:
    if not visited:
      distanceFromRed[neighbor] = distanceFromRed[current] + 1
      queue.push(neighbor)
```

### Phase 2: Calculate Expected Operations
```cpp
double total = 0

for each black node i:
  helpingNeighbors = 0
  
  for each neighbor j of node i:
    if j is red OR distance[j] < distance[i]:
      helpingNeighbors++
  
  if helpingNeighbors > 0:
    total += degree[i] / helpingNeighbors

return total
```

## Key Insights

1. **Greedy Dependency**: A black node only needs to be colored if it blocks other black nodes
2. **Distance as Priority**: Nodes closer to red are more likely to be colored naturally
3. **Random Walk Property**: Each operation is independent random walk toward red neighbors
4. **Linearity of Expectation**: Total expected value = sum of individual expected values

### Known Issue
Test case 2 produces 6.0 instead of expected 4.5. This suggests the formula might need:
- Iterative processing of nodes by layers
- Accounting for cascading effects where multiple black nodes depend on same red node
- Potential dependency between black node colorings not fully captured
