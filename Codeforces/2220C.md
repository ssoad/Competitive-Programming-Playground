# 2220C - Finding Grid Dimensions

## Problem Summary
Given two values p and q representing differences in horizontal and vertical dimensions, we need to find the number of horizontal segments (n) and segment length (m) for a grid such that the dimensional difference constraint is satisfied.

## Mathematical Model
A grid with n horizontal segments and segment length m has:
- Horizontal dimension: `h = (n + 1) * m`
- Vertical dimension: `v = (m + 1) * n`
- Constraint: `|h - v| ≤ p` and `h + v = p + 2*q`

From the constraint: `h + v = p + 2*q`, we derive:
- `(n + 1)*m + (m + 1)*n = p + 2*q`
- `m*(2*n + 1) + n = p + 2*q`
- `m = (p + 2*q - n) / (2*n + 1)`

## Algorithm
1. Try each value of n from 1 to 20000
2. Calculate m using the formula above
3. Check if m is a valid integer (divisible)
4. Verify the dimensional difference: `|h - v| ≤ p`
5. Return the first valid solution

**Time Complexity**: O(20000) per test case
**Space Complexity**: O(1)

## Solution Steps
```
for n = 1 to 20000:
  numerator = (p + 2*q) - n
  denominator = 2*n + 1
  
  if numerator > 0 and numerator % denominator == 0:
    m = numerator / denominator
    h = (n + 1) * m
    v = (m + 1) * n
    
    if |h - v| ≤ p:
      return (n, m)

return -1 (no solution found)
```

## Key Observations
- The denominat is always odd (2*n + 1), making divisibility checks straightforward
- The limit of 20000 is sufficient because for larger n, the mathematical constraints rarely yield valid solutions
- Dimension difference decreases as n increases, eventually failing the constraint

## Variable Naming
- `horizontalDiff` = p (input parameter)
- `verticalDiff` = q (input parameter)
- `numSegments` = n (horizontal segments)
- `segmentLength` = m (segment length)
- `combinedValue` = p + 2*q (combined constraint value)

## Example
For p=6, q=9:
- combinedValue = 6 + 2*9 = 24
- For n=2: m = (24-2)/(2*2+1) = 22/5 (not integer)
- Eventually finds a valid (n, m) pair
