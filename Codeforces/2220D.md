# 2220D - Three Positions

## Problem Summary
Interactive problem: find three positions i1, i2, i3 in a permutation of 2n+1 elements using queries. Each query returns the XOR of elements at specified positions. We need to find which elements are at positions i1, i2, i3 while minimizing queries.

## Key Insight
The problem uses parity checking with binary search:
- For a range [1...k], if the sum of k elements has different parity than k itself, one element differs from expected
- By binary searching on ranges, we can locate positions that break the expected parity pattern
- Three separate binary searches identify the three target positions

## Algorithm Overview

### Phase 1: Find i3 (first position from left with parity mismatch)
```
Binary search on range [1, N]:
  Check range [1, mid]
  If parity(query[1...mid]) != mid % 2:
    i3 = mid (candidate found)
    Search left half (hi = mid - 1)
  Else:
    Search right half (lo = mid + 1)
```

### Phase 2: Find i1 (position from right with parity mismatch)
```
Binary search on range [1, N]:
  Check range [mid, N]
  If parity(query[mid...N]) != (N-mid+1) % 2:
    i1 = mid (candidate found)
    Search right half (lo = mid + 1)
  Else:
    Search left half (hi = mid - 1)
```

### Phase 3: Find i2 (position between 1 and i3)
```
Binary search on range [1, i3-1]:
  Check range [1, mid] + {i3}
  If parity(query) != (mid+1) % 2:
    i2 = mid (candidate found)
    Search left half (hi = mid - 1)
  Else:
    Search right half (lo = mid + 1)
```

**Time Complexity**: O(3 * log N) = O(log N) queries per test case
**Space Complexity**: O(N)

## Implementation Details

### Query Function
```cpp
int sendQuery(const vector<int>& positions):
  - Output query format: "? size pos1 pos2 ..."
  - Flush output for interactive response
  - Read and return result
  - Exit if result is -1 (error case)
```

### Range Query Helper
```cpp
int querySingleRange(int start, int end):
  - Creates vector of consecutive integers [start...end]
  - Calls sendQuery()
  - Returns result
```

## Critical Points
1. **Flush after each query**: Required for interactive problems to get immediate responses
2. **Exact output format**: "?" for query, "!" for final answer
3. **Binary search boundaries**: Each phase searches different ranges and conditions
4. **Parity logic**: Correctly handling expected vs actual parity differences

## Variable Naming
- `testCases` = number of test cases
- `totalSize` = 2*n+1
- `posI1/I2/I3` = target positions
- `lo/hi` = binary search range bounds  
- `parity` = actual parity from query
- `expectedParity` = theoretical parity
- `closerNeighbors` = binary search steps
